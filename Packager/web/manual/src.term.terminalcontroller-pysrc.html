 56      """ 
 57      A class that can be used to portably generate formatted output 
 58      to a terminal. 
 59   
 60      See http://igraph.org/python/doc/igraph.app.shell.TerminalController-class.html 
 61   
 62      `TerminalController` defines a set of instance variables whose 
 63      values are initialized to the control sequence necessary to 
 64      perform a given action.  These can be simply included in normal 
 65      output to the terminal: 
 66   
 67      >>> term = TerminalController() 
 68      >>> print 'This is '+term.GREEN+'green'+term.NORMAL 
 69   
 70      Alternatively, the `render()` method can used, which replaces 
 71      '${action}' with the string required to perform 'action': 
 72   
 73      >>> term = TerminalController() 
 74      >>> print term.render('This is ${GREEN}green${NORMAL}') 
 75   
 76      If the terminal doesn't support a given action, then the value of 
 77      the corresponding instance variable will be set to ''.  As a 
 78      result, the above code will still work on terminals that do not 
 79      support color, except that their output will not be colored. 
 80      Also, this means that you can test whether the terminal supports a 
 81      given action by simply testing the truth value of the 
 82      corresponding instance variable: 
 83   
 84      >>> term = TerminalController() 
 85      >>> if term.CLEAR_SCREEN: 
 86      ...     print 'This terminal supports clearning the screen.' 
 87   
 88      Finally, if the width and height of the terminal are known, then 
 89      they will be stored in the `COLS` and `LINES` attributes.
